快捷组合Everything+AutoHotkey使用小记

继上次看的善用佳软在中科大的报告，除了主要的TC介绍外，还额外介绍了几个提高效率的软件，其中就有Everything和AutoHotkey。

Everything设置相对简单，请参看善用佳软的详细介绍“Everything:速度最快的文件名搜索工具”。除了首次打开时，略显卡顿之外（项目工程文件超多，复制任何一个工程都在小时级别，见谅！），之后再打开都是所见所得，速度嗖嗖的。当然安装中文包，看的会顺眼点。

重点说下AutoHotkey，先看善用佳软的介绍“AutoHotkey 0级入门教程:让重复工作一键完成”。安装完成后运行AutoHotkey.exe（还有AutoHotkeyA32.exe和AutoHotkeyU32.exe），右键选择Edit This Script，打开一个名叫AutoHotkey.ahk的文件（是否想到此时可以用Everything搜索到这个文件，此文件在我的文档目录下）。简单翻译下里面的注释：

这个script（指AutoHotkey.ahk）的文件名和路径都是特殊的，因为当你直接启动程序时，会自动加载这个文件。同样任何以.ahk结尾的文本文件都可以通过双击来加载。你可以在任意目录下创建任意多的.ahk文件。你也可以同时运行多个.ahk文件，每个文件都会有各自的托盘图标。可以理解为，将自己定义的快捷键方式放在该文件的后面，因为每次启动AHK都会加载该文件，而不选择重新新建一个，每次都双击去打开运行，当然测试阶段可能那样更方便。好，说重点：组合键快速打开Everything，比如Win+a,要求当Everything已处于激活状态（指在最上层），最小化Everything窗口；当Everything最小化或者在后台或者根本没启动时，打开并激活。根据这个要求，找了N多博文，甚至Google，仍没得到有效解决，最后只能硬着头皮看AHK英文帮助文档，发现不过几分钟就理解并实现了（不是炫耀，英文帮助文档内容全面，且思路清晰，只要耐心点很快就能找到想要的）。

; Hotkeys for SeattleZhu at 20130716


Activate(p)
{
  IfWinActive, ahk_exe %p%
  {
    WinMinimize
    return
  }
  IfWinExist, ahk_exe %p%
  {
    WinShow
    WinActivate
    return 1
  }
  return 0
}


ActivateAndOpen(t,p)
{
  if Activate(p)==0
  {
    Run %p%
    WinActivate
    return
  }
}


; Program in common use
#a::ActivateAndOpen(“Everything”,”D:\MyProgram\Everything\Everything.exe”)
#c::Run cmd.exe


; Common Website
#g::Run https://www.google.com/
#b::Run www.baidu.com


; Common file-parth
!b::Run E:\百度云

注：这段代码是结合了“AutoHotkey+Everything提升工作效率”一文所示代码，直接使用发现总是不如意，看帮助文档后发现关键问题出在WinTitle上，原代码是用Title来确认是否已激活或打开程序，而程序Title一直没找到（囧），但文档提示AHK[v1.1.01+]版本支持ahk_exe方式确认，既然Run命令要用到exe文件路径，索性合并在一起岂不是方便，于是就有了新代码。
